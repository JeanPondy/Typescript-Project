Typescript

01. Einleitung

TypeScript ist eine von Microsoft entwickelte Programmiersprache, die 2012 eingeführt wurde. Sie ist eine Erweiterung von JavaScript und bietet statische Typisierung, was bedeutet, dass Variablen, Funktionen und Parameter spezifische Datentypen zugewiesen werden können. Dies ermöglicht eine frühzeitige Fehlererkennung und verbessert die Codequalität sowie die Lesbarkeit des Codes
Vorteile und Einsatzgebiete von TypeScript
. Statische Typisierung: Im Gegensatz zu JavaScript, das dynamisch typisiert ist, erlaubt TypeScript die Deklaration von Datentypen, was die Fehleranalyse verbessert und die Entwicklung beschleunigt.
. Integration mit JavaScript: TypeScript ist ein Superset von JavaScript, was bedeutet, dass jeder gültige JavaScript-Code auch in TypeScript funktioniert. Dies erleichtert die schrittweise Umstellung von bestehenden Projekten auf TypeScript.
. Verbesserte Codequalität: Durch die statische Typisierung und zusätzliche Funktionen wie Klassen und Module wird der Code wartbarer und skalierbarer.
. Anwendungsbereiche: TypeScript wird häufig in großen Webentwicklungsprojekten eingesetzt, sowohl im Frontend als auch im Backend mit Node.js. Es ist auch in der Cross-Plattform-Entwicklung, z.B. mit React Native, und in der Spieleentwicklung nützlich.
Verwendung von TypeScript
Um TypeScript in einem Projekt zu verwenden, muss der TypeScript-Compiler installiert werden. Dies kann mit npm, einem Paketmanager für JavaScript, erfolgen. Der folgende Befehl installiert TypeScript als Entwicklungsabhängigkeit: npm install -D typescript
Nach der Installation kann der Compiler mit dem Befehl npx tsc aufgerufen werden, um TypeScript-Dateien in JavaScript zu kompilieren. TypeScript eignet sich besonders für Projekte, die eine hohe Codequalität und Wartbarkeit erfordern, und ist ideal für Entwickler, die von den Vorteilen der statischen Typisierung profitieren möchten, ohne auf die Flexibilität von JavaScript verzichten zu müssen.

06. literals und tupels

literals
Literals in TypeScript sind feste, unveränderliche Werte, die direkt im Code angegeben werden.
Es gibt verschiedene Arten von Literals, z. B.
Zeichenketten, Zahlen, boolesche Werte und spezielle Typen.
Beispiele:
. String Literal: Ein Text, der in Anführungszeichen steht.
let greeting: "Hallo" = "Hallo";
.Number Literal: Eine konkrete Zahl.
let age: 30 = 30;
. Boolean Literal: Entweder true oder false.
let isActive: true = true;
In TypeScript können Literals auch verwendet werden, um bestimmte Werte zu erzwingen. Wenn du z. B. eine Variable hast, die nur bestimmte Werte annehmen darf, kannst du dies mit Literaltypen festlegen:
type Direction = "links" | "rechts" | "oben" | "unten";
let move: Direction;
move = "links"; // gültig
move = "unten"; // gültig
move = "vorwärts"; // Fehler: "vorwärts" ist kein gültiger Wert

Tupel in TypeScript
Ein Tupel ist eine spezielle Art von Array, bei dem die Anzahl der Elemente und deren Typen festgelegt sind. Das bedeutet, dass ein Tupel genau eine bestimmte Anzahl von Werten eines bestimmten Typs enthält, und jeder Wert im Tupel hat einen definierten Typ.
let user: [string, number];
user = ["Alice", 30]; // gültig
user = [30, "Alice"]; // Fehler: falsche Reihenfolge der Typen
In diesem Beispiel ist user ein Tupel, das genau zwei Werte enthält:
1. Ein string (Name der Person)
2. Eine number (Alter der Person)
Tupel sind nützlich, wenn du eine feste Struktur brauchst, z. B. um Daten mit einer festen Anzahl von Attributen zu speichern, aber du dennoch sicherstellen möchtest, dass die Typen an der richtigen Stelle sind.
